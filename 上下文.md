#### 语义聚类引擎

| 组件 | 技术 | 状态 |
|:---|:---|:---|
| 文本向量化 | BAAI/bge-small-zh-v1.5 (512维) | 🟢 生产 |
| 向量索引 | FAISS IVF-Flat + IndexIDMap2 | 🟢 生产 |
| 自适应阈值 | 双层 Otsu (P75 过滤 + P90 合并) | 🟢 生产 |
| 图聚类 | Union-Find + 簇内复核 | 🟢 生产 |
| 标题注入 | canonical_title 精确匹配防漏召 | 🟢 生产 |
| Jaccard 后备 | IDF 加权 + 同义词扩展 + 实体惩罚 | 🟡 Fallback |

#### 评分算法（生产路径实际生效）

| 算法 | 位置 | 公式简述 | 状态 |
|:---|:---|:---|:---|
| 文章热度分 | events.py `_score_article()` | rank_bonus + heat/10000 + source_bonus | 🟢 核心 |
| 事件热度分 | topics.py `_event_score()` | 文章数×0.45 + 平台数×1.2 + 时效衰减 | 🟢 核心 |
| 主题匹配分 | topics.py `score_topic()` | 完整匹配14 > 关键词9 > 分词4 · 隐式×0.55 | 🟢 核心 |
| 复合相似度 | semantic_index.py `_composite_similarity()` | 0.6×cos + 0.25×time_decay + 0.15×跨平台 | 🟡 仅诊断 |
| IDF Jaccard | events.py `_idf_weighted_jaccard()` | Σ(共享词IDF) / Σ(并集词IDF) | 🟡 Fallback |
| 关键词提取 | keyword_extraction.py | TF-IDF + TextRank (damping=0.85) | 🟢 核心 |

> **注意**：
> - 复合相似度仅写入 `relation_score` 用于排序/诊断，**不参与 Union-Find 聚类合并**。聚类合并的判据是**纯 cosine + 双层 Otsu 自适应阈值**。
> - `routes.py` 中的 `_article_result_score()` 和 `_dedupe_payload_by_title()` 为死代码（已定义但未被调用）。
> - `semantic_cluster.py` 中的 `cluster_articles_semantic()` 为早期实现，仅在评测脚本中作为对比基线使用，不被生产路径调用。

## 三、云部署极简流程

```bash
# 1. 传源码 zip 到服务器，解压
unzip 2026042001-源码.zip -d /opt/yujing && cd /opt/yujing

# 2. 安装依赖
pip install -r requirements.txt

# 3. 编辑 .env
#    - 填 DEEPSEEK_API_KEY
#    - 设 EMBED_USE_MIRROR=1（国内服务器）
#    - 删 ALERT_DEBUG=1 和 BRIEF_INTERVAL_MINUTES=1

# 4. 启动（一条命令，MeiliSearch 自动拉起）
python -m app.main

# 5. 访问 http://服务器IP:8000
```

### 前端配置（如需远程访问）

当前 26 处 API 调用硬编码了 `http://localhost:8000`。两种解决方案：

1. **同机访问**：不改，直接用 localhost
2. **远程访问**：将 `http://localhost:8000` 全局替换为 `buildApiUrl()` 调用，通过 `VITE_API_BASE_URL` 环境变量控制

## 四、评测体系

### 聚类评测 (evaluate_clustering.py)

- Pair Precision / Recall / F1
- ARI / NMI / V-measure（需闭集标注）
- Smoke test：取 200 篇对比 Jaccard vs FAISS

### Agent 评测 (eval_agent.py)

- 20 条标准 query，覆盖 5 类场景
- 工具调用准确率、首次命中率
- 延迟统计 (P50/P90/P99)
- 人工 task_completion_rate 补充
