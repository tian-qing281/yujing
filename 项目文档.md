# 舆镜 YuJing 项目说明文档

## 1. 项目定位

`yujing` 是一款全平台舆情监控与 AI 研判系统（舆镜 YuJing）。系统通过对多平台热榜的实时采集、语义化聚类与 AI 增强分析，建立起从"海量噪声"到"决策结论"的处理链条。

### 核心能力体系：
-   **信号层 (Article)**: 实时监控全网 8 个核心权重平台的热榜数据。
-   **事件层 (Event)**: 利用 TF-IDF + 归一化标题聚类算法，将碎片信号收敛为事件单元。
-   **研判层 (AI Analysis)**: 基于 SSE 流式架构，驱动 DeepSeek LLM 结合实时内容输出深度分析报告。
-   **早报系统 (Morning Brief)**: 定时生成每日舆情早报，支持调试模式（间隔触发），前端自动轮询展示。
-   **异动告警 (Alert)**: 实时扫描高热事件，三级告警（critical/warning/info），前端自动弹出通知。
-   **PDF 导出**: 文章、事件、早报、AI 对话均可一键导出 PDF 报告（fpdf2 + 中文字体）。
-   **情感分析**: 本地 BERT 模型 (Johnson8187/Chinese-Emotion) 提供 8 维情绪分类，离线规则兜底。

## 2. 核心技术实现

### 2.1 全域深度采集引擎
已稳定接入 **8 个** 信源，通过 APScheduler 定时调度（默认 120s 间隔），24/7 不间断同步：
-   **综合门户**: 微博热搜、百度热搜、今日头条、知乎热榜。
-   **专业财经**: 财联社电报、华尔街见闻、澎湃新闻热榜。
-   **视频态势**: 哔哩哔哩全站实时热榜。

采集细节：
-   **正文抽取**: 通过 Jina Reader 将目标页面转为 Markdown 正文，针对头条、B站、知乎等平台有专用解析逻辑。
-   **Cookie 凭据**: 微博、百度、头条、B站、知乎 5 个平台需要 Cookie 认证，后 3 个为公开 API。
-   **共享连接池**: 所有爬虫共享 httpx AsyncClient，线程池隔离 DB 写操作。

### 2.2 事件聚合与可视化
-   **归一化聚类**: 基于 Canonical Title（正则去噪后的标题）+ Jaccard 分词相似度，合并阈值 0.22，归一化标题一致加 0.42 强制合并。
-   **事件态势分析**: 利用 **ECharts 6** 深度定制的可视化模态框，提供：
    -   **热度指数**: 时间序列下的态势演化趋势图。
    -   **密度谱系**: 时间轴上的情报发布分布。
    -   **信源权重**: 跨平台的声量占比饼图。
    -   **关键词图谱**: 基于 jieba TF-IDF 分词的核心词提取与展示。
    -   **情感雷达**: 8 维情绪分析雷达图（BERT 模型 + 规则兜底）。

### 2.3 交互体验
-   **搜索翻页**: 采用 **9 宫格布局**，兼顾视觉稳定性与信息承载量。
-   **中断控制**: 集成 `AbortController`，自动打断过期网络请求，消除 UI 竞态。
-   **混合检索**: **MeiliSearch 全文检索** + 后端 **12 分制逻辑重排**（标题全词 +12、关键词 +8、正文 +5）。
-   **统一搜索**: 一次查询同时返回事件、文章、时间轴线、平台分布。
-   **AI 对话**: 支持单篇文章上下文对话和通用 AI 研判对话。

## 3. 项目架构

### 3.1 目录结构
```
yujing/
├── app/                    # Python 后端
│   ├── api/routes.py       # FastAPI 路由 (~2570行)
│   ├── api/proxy.py        # 媒体透传代理（防盗链）
│   ├── crawler/            # 爬虫引擎
│   │   ├── sources/        # 8 个平台爬虫（继承 BaseSource）
│   │   ├── reader.py       # 正文抽取（Jina + 各平台专用解析）
│   │   ├── scheduler.py    # APScheduler 定时调度 + 早报定时生成
│   │   └── cookies/        # 平台 Cookie 凭据存储
│   ├── services/           # 业务逻辑层
│   │   ├── events.py       # 事件聚合（TF-IDF + 相似度聚类）
│   │   ├── emotion.py      # BERT 情感分析引擎
│   │   ├── search_engine.py # MeiliSearch 封装
│   │   └── topics.py       # 主题聚合（后台服务，前端无独立入口）
│   ├── database.py         # SQLAlchemy 模型定义
│   ├── llm.py              # DeepSeek LLM 封装
│   ├── schemas.py          # Pydantic 响应模型
│   └── main.py             # 应用入口 + MeiliSearch 自动启动
├── yujing-ui/             # Vue 3 前端
│   └── src/
│       ├── App.vue         # 主视图（单文件组件）
│       └── components/     # UI 组件集
├── runtime/                # 运行时资产
│   ├── db/hongsou.db       # SQLite 数据库
│   └── meili/              # MeiliSearch 引擎 + 索引数据
└── .env                    # 环境配置
```

### 3.2 技术栈

**后端**:
| 组件 | 技术 | 说明 |
|:---|:---|:---|
| Web 框架 | FastAPI + Uvicorn | 端口 8000 |
| 数据库 | SQLite + SQLAlchemy | WAL 模式，30s 超时 |
| 搜索引擎 | MeiliSearch | 端口 7700，自动启动 |
| AI/LLM | DeepSeek API (OpenAI 兼容) | 通过 LangChain 调用，SSE 流式输出 |
| NLP | jieba + scikit-learn | TF-IDF 分词 + 事件聚类 |
| 情感分析 | BERT (Johnson8187/Chinese-Emotion) | 8 类情绪，后台异步加载，离线规则兜底 |
| 爬虫调度 | APScheduler | AsyncIOScheduler，120s 间隔 |
| PDF 生成 | fpdf2 | 中文字体支持（微软雅黑） |
| HTTP 客户端 | httpx | 异步，共享连接池 |

**前端**:
| 组件 | 技术 | 说明 |
|:---|:---|:---|
| 框架 | Vue 3 (Composition API) | 单页面，无路由 |
| 构建 | Vite 8 | 开发端口 5173 |
| UI | TailwindCSS 3 + DaisyUI 5 | 自定义 hongsou 主题 |
| 图表 | ECharts 6 | 懒加载预热 |
| Markdown | marked | AI 总结渲染 |
| 设计语言 | hs-panel / hs-pill / hs-command-button | 自定义组件类 |

### 3.3 数据模型
-   **Article**: 基础资讯条目（source_id, title, url, rank, extra_info, ai_summary, ai_sentiment, content）
-   **Event**: 聚合事件（title, summary, keywords, sentiment, article_count, platform_count）
-   **EventArticle**: 事件-文章关联表（relation_score, is_primary）

### 3.4 前端组件
| 组件 | 说明 |
|:---|:---|
| App.vue | 主视图，数据获取、状态管理、SSE 对接 |
