# 舆镜 YuJing 部署手册

本手册指导如何从零开始搭建并运行 `yujing` 舆情研判工作台。

---

## 1. 运行环境要求

-   **操作系统**: Windows 10 / 11 (64位)
-   **Python**: 3.11+（推荐 3.13）
-   **Node.js**: 20+（推荐 24）
-   **搜索引擎**: MeiliSearch（已内置二进制文件于 `runtime/meili/`）
-   **BERT 模型**: Johnson8187/Chinese-Emotion（可选，需提前下载到本地 HuggingFace 缓存目录）

---

## 2. 快速部署步骤

### 2.1 依赖安装
1.  **后端环境**:
    ```powershell
    python -m venv .venv
    .\.venv\Scripts\activate
    pip install -r requirements.txt
    ```
2.  **前端环境**:
    ```powershell
    cd yujing-ui
    npm install
    ```

### 2.2 环境变量配置
复制项目根目录下的 `.env.example` 为 `.env`。

**必须配置项**:
| 变量 | 说明 | 示例 |
|:---|:---|:---|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | `sk-...` |
| `DEEPSEEK_MODEL` | 模型名称 | `deepseek-chat` |

**可选配置项**（一般无需修改）:
| 变量 | 默认值 | 说明 |
|:---|:---|:---|
| `DEEPSEEK_API_BASE` | `https://api.deepseek.com` | API 基地址 |
| `DATABASE_PATH` | `runtime/db/yujing.db` | 数据库路径 |
| `MEILI_BINARY_PATH` | `runtime/meili/meilisearch-windows-amd64.exe` | MeiliSearch 二进制 |
| `MEILI_DB_PATH` | `runtime/meili/data.ms` | MeiliSearch 索引目录 |
| `MEILI_MASTER_KEY` | 内置默认值 | MeiliSearch 密钥 |
| `CORS_ALLOW_ORIGINS` | `http://localhost:5173` | 前端跨域白名单 |

**早报与告警配置**:
| 变量 | 默认值 | 说明 |
|:---|:---|:---|
| `BRIEF_HOUR` | `8` | 早报定时生成-小时 (24h) |
| `BRIEF_MINUTE` | `0` | 早报定时生成-分钟 |
| `BRIEF_INTERVAL_MINUTES` | (空) | 调试用：设为正整数则按间隔触发早报（如 `2` = 每2分钟） |
| `ALERT_THRESHOLD` | `3` | 异动告警阈值：事件关联文章数 >= N 触发告警 |
| `ALERT_WINDOW_HOURS` | `2` | 告警检测时间窗口（小时） |
| `ALERT_DEBUG` | (空) | 设为 `1` 开启告警调试模式（阈值降为1，不限时间窗口） |

## 3. 启动指令

### 3.1 后端服务
在项目根目录下运行：
```powershell
.\.venv\Scripts\activate
python -m app.main
```
> 系统启动后会自动检测并启动 MeiliSearch 引擎（端口 7700）。启动后自动开始爬虫调度和数据预热。

### 3.2 前端工作台
在 `yujing-ui` 目录下运行：
```powershell
npm run dev
```
访问 `http://localhost:5173` 即可使用。

### 3.3 前端构建（生产）
```powershell
cd yujing-ui
npm run build
```
产出位于 `yujing-ui/dist/`。

## 4. 关键资产路径

| 资产类型 | 物理路径 | 说明 |
|:---|:---|:---|
| 持久化数据库 | `runtime/db/yujing.db` | SQLite，存储文章、事件、分析记录 |
| 搜索索引 | `runtime/meili/data.ms/` | MeiliSearch 物理索引 |
| 登录凭据 | `app/crawler/cookies/` | 各平台 Cookie（JSON 格式） |
| MeiliSearch 引擎 | `runtime/meili/meilisearch-windows-amd64.exe` | 搜索引擎二进制 |
| 环境配置 | `.env` | API 密钥等敏感配置 |

## 5. Cookie 凭据管理

以下平台需要配置 Cookie 才能正常采集：

| 平台 | Cookie 文件名 | 获取方式 |
|:---|:---|:---|
| 微博热搜 | `weibo_hot_search.json` | 浏览器登录后导出 |
| 百度热搜 | `baidu_hot.json` | 浏览器登录后导出 |
| 今日头条 | `toutiao_hot.json` | 浏览器登录后导出 |
| B站热榜 | `bilibili_hot_video.json` | 浏览器登录后导出 |
| 知乎热榜 | `zhihu_hot_question.json` | 浏览器登录后导出 |

Cookie 也可通过前端"凭据管理"界面粘贴保存。后 3 个平台（财联社、华尔街见闻、澎湃）为公开 API，无需 Cookie。

## 6. 常见问题 (Troubleshooting)

1.  **搜索无结果**:
    -   检查 MeiliSearch 是否在运行（端口 7700）。
    -   重启后端，系统会自动重连索引引擎。
2.  **AI 分析/总结加载失败**:
    -   检查 `.env` 中的 `DEEPSEEK_API_KEY` 余额及网络连通性。
    -   特定文章报错可能是 Cookie 失效触发了反爬验证。
3.  **情感分析显示"规则兜底"**:
    -   BERT 模型 (Johnson8187/Chinese-Emotion) 需要提前下载到本地 HuggingFace 缓存。
    -   模型在后台异步加载，首次启动可能需要等待数秒。
4.  **头条/知乎内容抓取失败**:
    -   这两个平台为纯 JS 渲染，依赖 Jina Reader 提取 Markdown。
    -   如果 Jina 服务不可达，会返回抓取失败提示。
5.  **PDF 导出报错**:
    -   需要系统安装微软雅黑字体（`C:/Windows/Fonts/msyh.ttc`），Windows 默认已预装。
    -   确认 `fpdf2` 已安装：`pip install fpdf2`。
6.  **早报未自动弹出**:
    -   前端每 15 秒轮询早报状态，生成后自动展示 banner。
    -   调试时设置 `BRIEF_INTERVAL_MINUTES=1` 可每分钟触发一次早报生成。
7.  **异动告警无数据**:
    -   正常模式要求 2 小时内有事件更新且 `article_count >= 3`。
    -   调试时设置 `ALERT_DEBUG=1` 降低阈值为 1，无时间窗口限制。
